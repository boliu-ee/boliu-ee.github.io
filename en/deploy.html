Deployment & Custom Domain

This page is intended as your practical launch guide. The site is already structured as a GitHub Pages / Gitee Pages compatible static website.

GitHub Pages

  1. Create a repository named yourname.github.io
  2. Upload all website files to the repository root
  3. Settings → Pages → Deploy from branch → main / root
  4. Wait for the site URL to be published

Custom domain

Create a CNAME file in the repository root, for example:

www.yourdomain.com

Then configure the DNS records in your domain provider dashboard.

Gitee Pages

  1. Create a public repository and upload the same static files
  2. Enable Gitee Pages and trigger deployment manually
  3. Use it as the China mirror while keeping GitHub Pages as the primary site

My recommendation

  • Primary site: GitHub Pages
  • Mirror: Gitee Pages
  • Domain binding: point the domain to GitHub Pages
